## Still Needed

### 1. npx First-Class Support (from #1044, inspired by Vercel/Firecrawl)

**Status:** npm package `@iblai/cli` exists but needs polish.

**What to do:**
- Ensure `npx @iblai/cli startapp agent` works out of the box with zero prior setup
- The npm binary wrapper at `.iblai/npm/cli/bin/iblai.js` should download the platform binary on first run (like Prisma, Turbo)
- Add a `--yes` flag to all interactive prompts for CI/CD usage (Vercel pattern)
- `npx @iblai/cli` with no subcommand should print a helpful guide (not just `--help`), showing the most common next steps -- similar to Vercel's post-deploy guidance

### 2. `iblai` Bare Command -- Smart Default (inspired by Vercel)

**What to do:**
- Running `iblai` with no arguments should show a rich help screen (via `rich`) with:
  - The 3 command groups (startapp, add, builds) with descriptions
  - A "Getting Started" section with the 2-3 most common commands
  - Links to docs/skills
- This is better than a generic Click help message

### 3. `iblai add account` and `iblai add analytics` (from #1142)

**Status:** Skills exist (`/iblai-add-account`, `/iblai-add-analytics`) but the CLI generators may not exist yet.

**What to do:**
- Add `AddAccountGenerator` in `.iblai/iblai/generators/add_account.py`
- Add `AddAnalyticsGenerator` in `.iblai/iblai/generators/add_analytics.py`
- Register in `.iblai/iblai/commands/add.py`
- Add templates in `.iblai/iblai/templates/add/account/` and `add/analytics/`
- Add tests

### 4. Dual-Context CLAUDE.md (from #1142 comment 2)

**Status:** Already partially done -- root `CLAUDE.md` is for app developers, `.iblai/CLAUDE.md` is for CLI developers.

**What to do:**
- Make the separation more explicit in both files
- Root `CLAUDE.md` should start with: "This is for app developers. For CLI development, see `.iblai/CLAUDE.md`."
- `.iblai/CLAUDE.md` should start with: "This is for CLI developers. For app development, see the root `CLAUDE.md`."
- Consider a `iblai config set context cli-dev` command that symlinks the appropriate CLAUDE.md (lower priority)

### 5. .env Configuration Support (from #1044)

**Status:** `.iblai/iblai/config.py` already supports `.env` file loading.

**What to do:**
- Document `.env` file support prominently in the README
- Support `.env.development`, `.env.staging`, `.env.production` stages
- `iblai config show` -- print current effective configuration
- `iblai config set <key> <value>` -- write to `.env`

### 6. Vercel-Inspired Patterns

**Post-action guidance:**
- After `iblai startapp agent`, print the logical next steps (cd, install, configure, dev)
- After `iblai add auth`, print what was created and what to do next
- This should be a `rich` panel at the end of each command

**Framework auto-detection:**
- `iblai add` already detects Next.js projects via `project_detector.py`
- Enhance to detect package manager, existing components, and suggest what to add next

**`iblai open`:**
- Open the iblai.app dashboard for the current tenant (reads `NEXT_PUBLIC_MAIN_TENANT_KEY` from .env.local)
- `iblai open docs` -- open the docs
- `iblai open app` -- open localhost:3000

### 7. Firecrawl-Inspired Patterns

**Agent-native onboarding:**
- `iblai init` -- configure the current project for AI-assisted development with ibl.ai
  - Adds `.mcp.json` pointing to `@iblai/mcp`
  - Adds `.claude/skills/` with app development skills
  - Adds `.opencode/skills/` and `.cursor/rules/`
  - This is essentially what `iblai add mcp` does, but named more intuitively

**URL-as-argument (future):**
- `iblai add https://shadcnspace.com/blocks/hero-01` -- add a shadcnspace block by URL
- Wraps `npx shadcn@latest add` but applies ibl.ai brand values automatically

### 8. shadcn/ui Theme Alignment (from #1142 comment 1)

**What to do:**
- Generated apps should include a shared CSS variables file that both ibl.ai SDK components and shadcn/ui consume
- The `components.json` generated by the CLI already configures shadcn/ui with `new-york` style and CSS variables
- Add a `tailwind.config` (or CSS) preset that maps ibl.ai brand tokens to shadcn CSS variables
- Document this in the README and in the `/iblai-add-shadcn-component` skill
- Test that mixing ibl.ai + shadcn components in the same page is visually seamless

### 9. App Store Submission Tools (from #1044)

**What to do:**
- `iblai builds screenshots` -- generate marketing screenshots for app store submissions
- `iblai builds submit ios` -- guide through iOS App Store submission
- `iblai builds submit android` -- guide through Google Play submission
- Consider integrating with [fastlane](https://fastlane.tools/) for automation
- Lower priority -- Tauri builds are the foundation, this is the polish

## Relationship to iblai/vibe

The [vibe](https://github.com/iblai/vibe) repository consumes this CLI. It provides:
- A README that sells the value proposition to vibe coders
- A CLAUDE.md that primes AI assistants for vibe coding with ibl.ai
- 5 Claude Code skills (`/vibe-new-app`, `/vibe-add-feature`, `/vibe-deploy`, `/vibe-customize`, `/vibe-connect-backend`)
- `.mcp.json` pointing to `@iblai/mcp`

The vibe repo is the "why" and "what"; this CLI is the "how". Simplifications here directly improve the vibe coding experience there.
